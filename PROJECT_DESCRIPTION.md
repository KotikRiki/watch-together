# Watch Together — Описание проекта, проблемы и пожелания

## Что это за проект
Веб-приложение для совместного просмотра видео в реальном времени. Пользователи создают комнату по коду/ссылке, смотрят видео вместе с синхронизированным воспроизведением, общаются в чате, отправляют стикеры, могут позвонить по видео/аудио.

## Технологии
- **Клиент**: React 19 + TypeScript + Vite 8 + Tailwind CSS 4 + Socket.io-client
- **Сервер**: Express 4 + TypeScript + Socket.io + PostgreSQL (Render Free) + JSON fallback
- **Деплой**: Render.com Free tier (512MB RAM, 100GB bandwidth/мес)
- **Репо**: https://github.com/KotikRiki/watch-together

## ИЗВЕСТНЫЕ ПРОБЛЕМЫ

### 1. Синхронизация загруженных файлов (КРИТИЧНО)
- Heartbeat каждые 3 секунды делает seek() к времени хоста
- Для YouTube это незаметно (плеер плавно корректирует)
- Для загруженных .mp4 файлов — каждый seek вызывает audio glitch, звук "двоится"
- Порог дрифта увеличен до 8 секунд для файлов, но проблема не решена полностью
- **Нужно**: более умная синхронизация для HTML5 <video> — плавная коррекция без seek

### 2. Работают только YouTube и RuTube
- **YouTube**: работает через IFrame API (включая поиск рекламы)
- **RuTube**: работает через postMessage API
- **VK Video**: НЕ работает — wrapper не инициализируется, VK изменил API
- **Загрузка файлов**: работает, но с проблемами синхронизации (см. п.1)
- **Нужно**: починить VK Video или найти альтернативу

### 3. Админ-панель (/admin)
- Некоторые данные не сохраняются между перезагрузками (ephemeral disk на Render)
- История видео и watch-time хранятся в PostgreSQL, но UI не удобный
- Нет возможности управлять стикерпаками из админки
- Нет возможности удалять/редактировать комнаты
- Нет экспорта данных
- **Нужно**: более удобный UI, CRUD операции, визуализация данных

### 4. Чат и стикеры
- Telegram стикеры загружаются через Bot API, проксируются через сервер
- В панели стикеров показываются только эмодзи (видео-превью не грузятся из-за медленного прокси на Render Free)
- В чате стикеры отображаются как autoplay видео — работает
- Нет поддержки GIF
- Нет ответа на сообщения (reply)
- Нет уведомления о новых сообщениях (badge/push)
- **Нужно**: reply, уведомления, GIF, стикерпаки из админки

### 5. Видеозвонок (WebRTC)
- Работает, но без TURN сервера — не работает за NAT/firewall
- Нет screen sharing
- **Нужно**: TURN сервер, screen sharing, запись

### 6. Общее
- Render Free: 512MB RAM, 100GB bandwidth/мес — быстро заканчивается
- PostgreSQL на Render Free истекает через 30 дней неактивности
- Нет PWA (не устанавливается на телефон)
- Нет offline-режима
- Нет темной/светлой темы (только тёмная)
- Нет языковой поддержки (только русский)
